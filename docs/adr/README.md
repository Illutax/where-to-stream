(Kontext, Optionen, Trade-offs) — nicht **wie** das System implementiert ist. Format:
[Nygard-ADR](https://cognitect.com/blog/2011/11/15/documenting-architecture-decisions), analog
zum `adr`-Skill der Team-Doku.
